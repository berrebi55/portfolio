Projet Véliko : Déploiement & Infrastructure

Déploiement et configuration d’une application Symfony sur une architecture distribuée.

Contexte et Objectifs

Le projet BTS-SIO-G12-2025-VELIKO-Web est une application Symfony exploitant TailwindCSS et Stimulus en front-end, ainsi que MariaDB et une Mock API en back-end. Le déploiement s'effectue sur une infrastructure composée de trois serveurs distincts.

1. Serveur 1 : Application Web

  • Fonction : Héberger l'application Symfony.
  • Technologies : Apache2, PHP, Composer, Symfony CLI.
  • Problèmes : Permissions incorrectes, configuration des variables d’environnement, dépendances manquantes.
  • Solutions : Configuration des permissions, gestion des variables d’environnement avec `.env.local`, installation des dépendances avec Composer.

2. Serveur 2 : Services Associés

  • Fonction : Héberger des services Docker tels que Mailhog (simulateur SMTP) et Mock API.
  • Problèmes : Ports réseau bloqués, variables d’environnement manquantes.
  • Solutions : Configuration des ports avec `docker-compose.yml`, gestion des variables avec un fichier `.env` dédié.

3. Serveur 3 : Base de Données

  • Fonction : Héberger MariaDB avec accès distant sécurisé.
  • Problèmes : Connexions distantes refusées, permissions utilisateur insuffisantes.
  • Solutions : Modification de `my.cnf` pour autoriser les connexions distantes, gestion des permissions avec `GRANT ALL PRIVILEGES`.

4. Configuration HTTPS et Sécurisation

  • Génération et installation d’un certificat SSL via Let’s Encrypt.
  • Configuration d’Apache avec redirection automatique HTTP → HTTPS.
  • Mise en place d’un pare-feu avec UFW pour restreindre les accès.
